You are a Mobile Systems Performance Research Architect.

Your task is to design the CROSS-PLATFORM BENCHMARK HARNESS
for mobile-systems-lab.

This system must empirically measure and compare:

- Kotlin (Android native)
- Swift (iOS native)
- Flutter
- React Native

Using the same canonical lab implementations.

----------------------------------------
SECTION 1 — DEFINE WHAT TO MEASURE
----------------------------------------

Design measurable metrics:

1. Cold start time
2. Warm start time
3. Memory footprint at idle
4. Memory footprint during sensor streaming
5. CPU usage during:
   - Idle
   - Accelerometer streaming
   - GPS streaming
6. Frame rendering stability (jank detection)
7. Sensor event throughput
8. Battery impact estimation
9. Background execution reliability
10. Config load latency

For each metric:
- Define measurement method
- Define sampling strategy
- Define normalization method
- Define tooling per platform

----------------------------------------

----------------------------------------

For each ecosystem document:

Android:
- adb shell commands

- battery historian

iOS:
- Instruments
- Xcode profiler
- energy log
- sysdiagnose

Flutter:
- flutter devtools
- performance overlay
- timeline events

React Native:
- Hermes profiling
- Flipper
- performance monitor

----------------------------------------
SECTION 3 — UNIFIED BENCHMARK PROTOCOL
----------------------------------------

Design a reproducible benchmark protocol:

1. Device preparation steps
2. Clean reboot
3. Install fresh debug build
4. Cold start measurement
5. Sensor activation for fixed duration
6. Data logging period
7. Export logs
8. Standardized result JSON output

Define:
BENCHMARK_RESULT.json schema

----------------------------------------

Design a normalization layer that converts:

Android logs
iOS logs
Flutter logs
React Native logs

UNIFIED_METRICS.json
